From Timeline to Telegram

How Huginn turns 80+ raw tweets into a curated hourly digest using a 4-layer scoring pipeline

The Problem

X's algorithmic timeline serves 60–80 tweets per fetch — a mix of viral posts, engagement bait, shop accounts, genuine insights, and everything in between. The original approach sent all of them to Claude Haiku and asked it to "pick the best ones."

This had three problems:

  1. No quality signals — engagement metrics were partially stripped during formatting, so Haiku had no way to distinguish a 50K-like post from a 3-like post
  2. Token waste — 80 tweets × ~100 tokens = 8K input tokens per run, mostly noise
  3. Inconsistent digests — without ranking signals, digest quality varied wildly between runs

The solution: a multi-layer scoring pipeline that runs before the LLM ever sees a tweet. Each layer adds an independent quality signal. By the time Haiku receives the tweets, they're pre-ranked and filtered to the top 30.

Two repos collaborate: huginn-jarvis handles fetching and scoring, muninn handles digest generation and Telegram delivery. They communicate through huginn's indexed collection.

L1 Engagement Scoring

Each tweet gets a composite engagement score using weights from X's open-sourced Heavy Ranker algorithm. The key insight: normalize by sqrt(views) to measure engagement rate, not raw volume. A niche tweet with 500 likes from 5K views ranks higher than a celebrity tweet with 1,000 likes from 1M views.

SignalWeightRationale
Retweets20xStrongest amplification — someone put their name on it
Replies13.5xIndicates discussion, controversy, or genuine interest
Bookmarks10xHigh-intent private save — "I'll come back to this"
Likes1xBaseline engagement (lowest signal, easiest action)
engagement = (likes·1 + retweets·20 + replies·13.5 + bookmarks·10) / √views

Content-type boosts stack multiplicatively:

  • Long-form notes: +20% — authors invested more effort
  • Quote tweets: +10% — commentary adds context
  • Media attachments: +5% — visual content
High engagement rate (ranks higher)
@karpathy: 500 likes, 50 RT, 20 replies from 5K views → score: 87.7
High raw numbers but low rate (ranks lower)
@celebrity: 1,000 likes, 20 RT from 1M views → score: 1.4

L2 Embedding-Based Interest Relevance

Not all high-engagement tweets are relevant. A viral meme might score high on engagement but low on relevance to AI and developer tools. Layer 2 uses the same multilingual-e5-base model as huginn's search index to compute cosine similarity between each tweet and 5 user-defined interest profiles.

L3 Author Graph Scoring

Layers 1 and 2 score individual tweets. Layer 3 asks a different question: is this author consistently high-signal? It builds a directed interaction graph from all accumulated tweet data and uses network analysis to score authors by structural importance and content quality.

How the graph is built

Nodes — every @handle seen in the timeline (1,877 unique authors from filenames, plus 580 quoted/mentioned-only handles = 2,457 total)
Edges — extracted from tweet markdown:
  • quote: > **Quoted @handle:** (weight: 3x)
  • mention: @handle in tweet body (weight: 1x)

Two algorithms, two signals

PageRank

Runs on the directed graph (α=0.85). Finds structurally important authors — accounts that many high-signal people quote or mention. @claudeai has the highest PageRank (1.000 normalized) because it's the most quoted account in the dataset.

Louvain Communities

Runs on the undirected projection. Discovers author clusters automatically — who interacts with whom. Found 1,240 communities, with the largest being an AI/Claude cluster (118 authors), tech industry (81), and ML/data (76).

author_score = 0.5 × pageranknorm + 0.5 × engagementnorm

Score Blending: Putting It All Together

Each layer produces a score in [0, 1]. The final combined_score is a weighted blend that evolved as layers were added:

Phase 1 engagement only
Phase 2 + relevance
Phase 3 (current) + author graph
Phase 4 (planned) + LLM reranker
PhaseFormulaStatus
1final = engagement_scoreSuperseded
2final = 0.50 × eng + 0.50 × relSuperseded
3final = 0.35 × eng + 0.35 × rel + 0.30 × authorCurrent
4LLM reranks top-N from phase 3Planned

Normalization

Each score must be in [0, 1] before blending so no single layer dominates:

  • Engagement: min(1.0, log1p(raw) / 6.2) — log scale compresses the long tail, 6.2 = log1p(500) is a fixed ceiling
  • Relevance: native [0, 1] from cosine similarity
  • Author: native [0, 1] from PageRank + engagement blend

When author scores aren't available (e.g., first run before graph is built), the scorer falls back to 2-way blending automatically.

The Digest Pipeline

Once tweets are scored and indexed, muninn's scheduler picks them up and turns them into a Telegram notification. Here's the complete sequence:

(hourly check) S->>W: checkX() W->>H: GET /api/collection/x-feed/documents H-->>W: Document list (IDs + URLs) W->>W: Filter by date

The scheduler runs every 60 seconds. The X watcher fires every hour to check for new high-signal tweets. Dedup ensures no tweet is ever sent twice.

The Haiku prompt

Haiku receives the top-30 tweets, pre-sorted by score, with this instruction:

Create a digest with two sections:
- **Top Picks** (3-5): most interesting/impactful, high-engagement content
  with a view-to-like ratio suggesting genuine interest
- **Also Notable** (up to 10): one-liner summaries

Filter out: spam, engagement bait, low-effort listicles, crypto shilling.
Tweets are pre-ranked by engagement score (highest engagement first).

What You Get: The Telegram Digest

Every hour, the watcher checks for new interesting tweets. When it finds them, the Markdown digest from Haiku is converted to Telegram-safe HTML (<b>, <i>, <a>) and delivered via a Grammy bot.

Cost & Efficiency

Before (all tweets to LLM)

  • ~80 tweets × 100 tokens = 8K input tokens
  • No quality signals for ranking
  • Inconsistent digest quality
  • All scoring done by Haiku (expensive)

After (pre-ranked pipeline)

  • 30 tweets × 60 tokens = 1.8K input tokens
  • 3-layer scoring with engagement, relevance, author
  • Consistently high-quality digests
  • Scoring: free (local Python + embeddings)
